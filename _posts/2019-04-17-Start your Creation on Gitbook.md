> [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) 📖 is a modern documentation 📃 format and toolchain using Git and Markdown. Supporting Markdown and AsciiDoc syntax, [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) is able to output HTML, PDF, eBook, etc.

## Git + Markdown

[Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) is not just a Markdown editing tool or a Git version management tool, it's more likely to be a document format conversion tool. 

There are many Markdown editors available on the market, such as Typora, MacDown, Bear, MarkdownPad, MarkdownX, JetBrains' IDE (plugins need to be installed), Atom, sketchbook, CSDN, and [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
)'s own [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) Editor.

Since you have noticed this blog, I believe you must have known the advantages (or elegance 😊) of  [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
). Based on that, I'm going to introduce the usage of [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) in a nutshell, to help you start your venture of creating!  🎉

## Requirements

Here I use `Github` + `Gitbook Pages` + `Typora ` to write a fitness manual 📓 as an example. (*You can also use other methods to do the same job, but this seems to be the simplest way I found to build pages on [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
).*)

## Personal Account

Open [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) website, you can see a page like this. Click `Get Started`  you can start your creations. Before documenting your thinking, you still need to create an account. **Here I strongly recommend using [Github](https://www.github.com/) to log in, for we will bind a repo with a book in the later.** There are some great tricks about writing on `Blog`, it's amazing! 🙃

![](https://upload-images.jianshu.io/upload_images/3220531-0b7e060dd1fa9fc4.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

## New Workspace

Go to your dashboard, and change the workspace name you want. For me, I want to create a workspace named `Fitness Manual`. Workspace serves like a collection. You are expected to put similar documents together in a certain workspace. It's a good manner!

Besides personal account, you can also create organizations by click `+` on the left top. Your friends, classmates, and colleagues can be in the same workspace in the organization account.

![](https://upload-images.jianshu.io/upload_images/3220531-6c6267e470f8be26.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

## New Book

Cut the cackle! Let' create a book! 😎

Let me first tell you the icons on the left top.

`Content` can overview your book process by offering current state, activity, as well as some drafts. 😐

![](https://upload-images.jianshu.io/upload_images/3220531-b473523b505ebe0f.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

`Share` is the public (if you set to public your book) version of your book. 😏

![](https://upload-images.jianshu.io/upload_images/3220531-89cfe9a4059449a3.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

`Customization` helps you customize your book, including icon, name, and domain name. Some themes, colors, and fonts are not free. 😛

![](https://upload-images.jianshu.io/upload_images/3220531-a5d6241b8806e05d.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

`Teams` is of little use in this part, it majorly exerts its effect in organization account. The team is the derivation of organization, like departments in colleges. So you can see them as *organizations in organizations*. 👨🏻‍💻

![](https://upload-images.jianshu.io/upload_images/3220531-81900a9596725521.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

**`Integration`** is a good part, for we can **integrate Github into [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
)**. Open the Github integration and link your repo 🔗 . The whole contents in that repo can be saved in your [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) automatically. The advantage of such trick is you can archive your articles twice. Some legacy parts are hard to adapt to the new version. Linking with [Github](https://www.github.com/) repository, you can easily immigrate your books to new version [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
).

![](https://upload-images.jianshu.io/upload_images/3220531-189f6f982f0ffc26.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

Initialization will help you add an `Introduction` in your new book.  Update your repo and add some articles on Github, effects will show on [Gitbook]([https://www.gitbook.com/](https://www.gitbook.com/)
) as well! 

![](https://upload-images.jianshu.io/upload_images/3220531-24bed00dca87f9d3.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)
