> Brian is a free, open source simulator for spiking neural networks. It is written in the Python programming language and is available on almost all platforms. We believe that a simulator should not only save the time of processors, but also the time of scientists. Brian is therefore designed to be easy to learn and use, highly flexible and easily extensible.

I am going to introduce this powerful tool with some basic demos about its features.

### Installation

We can use pip3 to install Brian2 easily.

```bash
pip3 install brian2
```

```bash
jupyer notebook
```

### Differential Equations

```python
tau = 10*ms
eqs = '''
dv/dt = (1-v)/tau : 1
'''

G = NeuronGroup(1, eqs, method='exact')
M = StateMonitor(G, 'v', record=0)

run(100*ms)

plot(M.t/ms, M.v[0], 'C0', label='Brian')
plot(M.t/ms, 1-exp(-M.t/tau), 'C1--',label='Analytic')
xlabel('Time (ms)')
ylabel('v')
legend();
```

![](https://tva1.sinaimg.cn/large/0082zybply1gc9wje70epj30aq07aweo.jpg)

First, we define a variable `tau` and differential equation `"dv/dt = (1-v)/tau : 1"`, where `: 1` is the SI unit of that variable. *Note that this is not the unit of the two sides of the equation (which would be `1/second`), but the unit of the variable defined by the equation, i.e. in this case v.*

Then we create a neuron group `G` with a neuron, using `eqs` equation and exact method. We set a state monitor `M` to keep track on `G`, where `record=0` means we record all values of `G`.

Afterward, we run the neuron for 100ms and plot it. When compared with analytic equation $1-e^{(-t/tau)}$, we find two lines overlap.

### Neuronal Characteristics

In this part, we are going to do something concerning neuronal characteristics, i.e. spikes and refractoriness. 

#### Spikes

Spikes, namely the mutations of neuronal value, are common in neurons. Let's see what's new in the following code.

```python
G = NeuronGroup(1, eqs, threshold='v>0.6', reset='v = 0.2', method='exact')

statemon = StateMonitor(G, 'v', record=0)
spikemon = SpikeMonitor(G)

run(80*ms)

plot(statemon.t/ms, statemon.v[0], c='C3', lw=3)
for t in spikemon.t:
    axvline(t/ms, ls='--', c='C4', lw=3)
xlabel('Time (ms)')
ylabel('v');
```

![](https://tva1.sinaimg.cn/large/0082zybply1gc9xttd98ij30aq07a0te.jpg)

This time we add to parameters to the `NeuronGroup` class, where `threshold='v>0.6', reset='v = 0.2'` means when $v>0.6$, we reset its value to $0.2$. We make two monitors, i.e. state monitor and spike monitor to keep track of these two behaviors and run this neuron for 80 ms. We use the `axvline` to draw the time of each spike.

#### Refractoriness

Another obsiquious characteristic of neuron is refractoriness, namely being unmanageable. We conduct extensive experiments to simulate refractoriness by Brian.

```python
import matplotlib.pyplot as plt

tau = 10*ms
eqs0 = '''
dv/dt = (1-v)/tau : 1
'''
eqs1 = '''
dv/dt = (1-v)/tau : 1 (unless refractory)
'''
eqs2 = '''
dv/dt = (1-v)/tau : 1
'''

G0 = NeuronGroup(1, eqs0, threshold='v>0.6', reset='v = 0.2', method='exact')
G1 = NeuronGroup(1, eqs1, threshold='v>0.6', reset='v = 0.2', refractory=10*ms, method='exact')
G2 = NeuronGroup(1, eqs2, threshold='v>0.6', reset='v = 0.2', refractory=10*ms, method='exact')


statemon0 = StateMonitor(G0, 'v', record=0)
spikemon0 = SpikeMonitor(G0)
statemon1 = StateMonitor(G1, 'v', record=0)
spikemon1 = SpikeMonitor(G1)
statemon2 = StateMonitor(G2, 'v', record=0)
spikemon2 = SpikeMonitor(G2)

run(80*ms)

plt.figure(figsize=(10, 10))
plt.subplot(2, 2, 1)
plt.title("G1")
plot(statemon1.t/ms, statemon1.v[0], c='C3')
for t in spikemon1.t:
    axvline(t/ms, ls='--', c='C4', lw=3)
axhline(0.6, ls=':', c='C2', lw=3)
xlabel('Time (ms)')
ylabel('v');

plt.subplot(2, 2, 2)
plt.title("G1")
plot(statemon1.t/ms, statemon1.v[0], c='C3')
for t in spikemon1.t:
    axvline(t/ms, ls='--', c='C4', lw=3)
axhline(0.6, ls=':', c='C2', lw=3)
xlabel('Time (ms)')
ylabel('v');

plt.subplot(2, 2, 3)
plt.title("G0")
plot(statemon0.t/ms, statemon0.v[0], c='C3')
for t in spikemon0.t:
    axvline(t/ms, ls='--', c='C4', lw=3)
axhline(0.6, ls=':', c='C2', lw=3)
xlabel('Time (ms)')
ylabel('v');

plt.subplot(2, 2, 4)
plt.title("G2")
plot(statemon2.t/ms, statemon2.v[0], c='C3')
for t in spikemon2.t:
    axvline(t/ms, ls='--', c='C4', lw=3)
axhline(0.6, ls=':', c='C2', lw=3)
xlabel('Time (ms)')
ylabel('v');
```

![](https://tva1.sinaimg.cn/large/0082zybply1gc9zz50vipj30gx0gswh7.jpg)

Here we create 3 neuron groups `G0`, `G1`, and `G2` (*of course you can create 3 neurons in only one neuron group*), where `G0` has no refractoriness, `G1` has `(unless refractory)` in its function, `G2` doesn't have  `(unless refractory)` in its function. Just like the experiment in the section Spike, we subplot the three neurons at position 2, 1 and 3, 4, respectively.

After observing `G1`, we find that when having `(unless refractory)` in its differential function, the refractoriness does not affect the slope and threshold of the value, just prolongs the neuron's respond for 10ms. 

While `G2`, which does not have `(unless refractory)`, breaks through the threshold of 0.6. Why? When the neuronal value rises to threshold (10ms or so in `G2`), the neuron fires a spike and will not fire anther spike before it get through the refractionariness period ($10+10$ ms).

### Mutiple Neurons

So far we've only been working with a single neuron. Let's do something interesting with multiple neurons.

#### Group Behavior 

Now, let's design a experiment to see the observation of a neuron group, i.e. mutiple neurons.

```python
start_scope()

N = 100
tau = 10*ms
eqs = '''
dv/dt = (2-v)/tau : 1
'''

G = NeuronGroup(N, eqs, threshold='v>1', reset='v=0', method='exact')
G.v = 'rand()'

spikemon = SpikeMonitor(G)

run(50*ms)

plt.figure(figsize=(10, 10))
plt.subplot(2, 2, 1)
plt.title("random 100 neurons - 50 ms run")
plot(spikemon.t/ms, spikemon.i, '.', c='C1')
xlabel('Time (ms)')
ylabel('Neuron index')


run(50*ms)  

plt.subplot(2, 2, 2)
plt.title("random 100 neurons - 100 ms run")
plot(spikemon.t/ms, spikemon.i, '.', c='C1')
xlabel('Time (ms)')
ylabel('Neuron index')


G = NeuronGroup(N, eqs, threshold='v>1', reset='v=0', method='exact')
G.v = 0

spikemon = SpikeMonitor(G)

run(50*ms)

plt.subplot(2, 2, 3)
plt.title("homo 100 neurons - 50 ms run")
plot(spikemon.t/ms, spikemon.i, '.', c='C1')
xlabel('Time (ms)')
ylabel('Neuron index')


N = 300

G = NeuronGroup(N, eqs, threshold='v>1', reset='v=0', method='exact')
G.v = 'rand()'

spikemon = SpikeMonitor(G)

run(50*ms)

plt.subplot(2, 2, 4)
plt.title("random 300 neurons - 50 ms run")
plot(spikemon.t/ms, spikemon.i, '.k', c='C1')
xlabel('Time (ms)')
ylabel('Neuron index')
```

![](https://tva1.sinaimg.cn/large/0082zybply1gca0xt54w5j30h00gs0x7.jpg)

*P.S. The code above might not be in a good coding style. But forget about this, I just want to show the results and to analyze multi-neurons behavior.*

In this experiment, I just initialize our neuron group with more neurons and try to find the effects of these parameters, i.e. the running time, the initilized value of neurons, the size of neuron group. Shown above, we see that:

(a $\leftrightarrow$ b) In a longer running time, neurons have more chance to spike;

(a $\leftrightarrow$ c) When initializing neurons with same value, they almost spike synchronously;

(a $\leftrightarrow$ d) Of course, we can get more  spikes if there are more neurons in the neuron group.

#### Equation Parameters

Some per-neuron parameters that don't have a differential equation attached to them. They are important because when making models of neurons, we include a random element to model the effect of various forms of neural noise. Let's see an example.

```python
# Per-neuron parameters
N = 50
tau = 10*ms
v0_max = 3.
duration = 1000*ms

eqs = '''
dv/dt = (v0-v)/tau : 1 (unless refractory)
v0 : 1
'''

G = NeuronGroup(N, eqs, threshold='v>1', reset='v=0', refractory=5*ms, method='exact')
M = SpikeMonitor(G)

G.v0 = 'i*v0_max/(N-1)'

run(duration)

figure(figsize=(10,10))
subplot(221)
plt.title("(A)")
plot(M.t/ms, M.i, '.', c='C2')
xlabel('Time (ms)')
ylabel('Neuron index')
subplot(222)
plt.title("(B)")
plot(G.v0, M.count/duration, c='C2', lw='4')
xlabel('v0')
ylabel('Firing rate (sp/s)')


# Stochastic neurons
N = 50
tau = 10*ms
v0_max = 3.
duration = 1000*ms
sigma = 0.2

eqs = '''
dv/dt = (v0-v)/tau+sigma*xi*tau**-0.5 : 1 (unless refractory)
v0 : 1
'''

G = NeuronGroup(N, eqs, threshold='v>1', reset='v=0', refractory=5*ms, method='euler')
M = SpikeMonitor(G)

G.v0 = 'i*v0_max/(N-1)'

run(duration)

subplot(223)
plt.title("(C)")
plot(M.t/ms, M.i, '.', c='C2')
xlabel('Time (ms)')
ylabel('Neuron index')
subplot(224)
plt.title("(D)")
plot(G.v0, M.count/duration, c='C2', lw=4)
xlabel('v0')
ylabel('Firing rate (sp/s)');
```

![](https://tva1.sinaimg.cn/large/0082zybply1gca2o8souvj30gu0gsju6.jpg)

Let first see the subplot (A) and (B) as an easy version of per-neuron parameter experiment. In this experiment, we use `V0` as a per-neuron parameter. For the sake of intuition, we refer to the previous analysis, i.e the original function of `dv/dt = (v0-v)/tau : 1` is $V0-e^{(-t/tau)}$. Considering that the threshold and the $V0_{max}$ in the initial value of neuron are all constants, neurons who have bigger indices ought to fire more spikes. Shown in (A) and (B), our assumption is validated.
